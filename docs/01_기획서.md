# 공동육아 플랫폼 기획서

## 서비스명 (가칭): 같이크자

> "우리 아이와 비슷한 또래 친구를 동네에서 쉽게 만나는 육아 모임 플랫폼"

---

## 1. 서비스 개요

### 1.1 문제 정의

- 비슷한 개월수 아이를 키우는 부모들이 주변에서 육아 친구를 찾기 어려움
- 놀이터/키즈카페에서 우연히 만나는 것 외에 체계적인 연결 수단이 없음
- 기존 맘카페는 온라인 커뮤니티일 뿐, 실제 만남으로 이어지기 어려움

### 1.2 해결 방안

- **지역 + 날짜/시간 + 개월수** 기반으로 육아 모임 "방"을 개설
- 조건이 맞는 부모가 방을 검색하고 참여 신청
- **자유 입장** 또는 **방장 수락** 방식으로 운영

### 1.3 타겟 유저

| 구분 | 설명 |
|------|------|
| 주 타겟 | 0~36개월 아이를 키우는 부모 (엄마/아빠) |
| 보조 타겟 | 37~72개월(미취학) 아이를 키우는 부모 |

## 2. 핵심 기능

### 2.1 방 (모임) 시스템

핵심 플로우: **방 생성 → 검색/발견 → 참여 신청 → 모임 진행 → 후기**

#### 방 생성

| 필드 | 필수 | 설명 |
|------|------|------|
| 제목 | O | 모임 제목 |
| 설명 | O | 모임 소개 |
| 지역 | O | 시/구/동 단위 선택 |
| 날짜 | O | 모임 날짜 (번개 모임은 오늘 가능, 일반은 30일 이내) |
| 시작 시간 | O | 모임 시작 시간 |
| 종료 시간 | X | 예상 종료 시간 |
| 대상 개월수 | O | 범위 지정 (예: 6~12개월) |
| 장소 유형 | O | 놀이터 / 키즈카페 / 파티룸 / 공원 / 기타 |
| 장소 상세 | X | 구체적 장소명, 주소 (**참여 확정자에게만 공개**) |
| 최대 인원 | O | 2~10명 |
| 입장 방식 | O | 자유 입장 / 승인 필요 |
| 참여 성별 | O | 전체 / 엄마만 / 아빠만 |
| 한부모 우대 | O | 무관 / 한부모만 |
| 번개 모임 여부 | O | 일반 / 번개 (24시간 이내 시작) |
| 준비물 체크리스트 | X | 추천 항목 토글 + 커스텀 입력 |
| 비용 | X | 무료 / 1인당 금액 |
| 태그 | X | #이유식교류 #산책 #놀이 등 |

#### 방 상태

```
모집중 → 모집완료 → 진행중 → 완료 → (자동 종료)
         ↘ 취소
```

#### 입장 방식

- **자유 입장**: 신청 즉시 참여 확정
- **승인 필요**: 방장이 프로필 확인 후 수락/거절

### 2.2 검색 및 필터

| 필터 | 설명 |
|------|------|
| 지역 | 내 동네 / 인근 동네 |
| 날짜 | 오늘 / 내일 / 이번 주 / 날짜 선택 |
| 개월수 | 내 아이 개월수 기준 자동 필터 |
| 장소 유형 | 놀이터 / 키즈카페 / 파티룸 등 |
| 비용 | 무료만 / 전체 |

> **규칙**: 약속 날짜가 지난 방은 목록에서 자동 제외 (조회 불가). 본인이 참여한 지난 모임은 마이페이지 > 지난 모임에서만 확인 가능.

### 2.3 네이버 지도 뷰

방 목록을 지도 위에서 탐색하는 기능

| 줌 레벨 | 표시 방식 | 설명 |
|---------|-----------|------|
| 축소 (광역) | 클러스터 (원형 뱃지) | 동네별 모집중인 방 개수를 원 안에 숫자로 표시 |
| 확대 (상세) | 핀 마커 | 개별 방의 약속 장소에 핀 표시, 탭하면 방 요약 팝업 |

- 네이버 지도 SDK (naver_map_plugin) 사용
- 방 생성 시 장소 좌표(위도/경도)를 네이버 Geocoding API로 변환하여 저장
- 클러스터 ↔ 핀 전환 기준: 줌 레벨 14 (동 단위 식별 가능 시점)

### 2.3 유저 프로필

| 필드 | 필수 | 설명 | 수정 |
|------|------|------|------|
| 닉네임 | O | 표시 이름 | 가능 |
| 프로필 사진 | X | 본인 사진 (선택) | 가능 |
| 지역 | O | 시/구/동 | 가능 |
| 부모 성별 | O | 엄마 / 아빠 (가입 시 선택) | **불가** |
| 한부모 가정 | O | 예 / 아니오 (가입 시 토글) | **불가** |
| 아이 정보 | O | 아래 상세 (최소 1명, 최대 5명) | 가능 |
| 자기소개 | X | 한줄 소개 | 가능 |

> **한부모 가정 정보 노출 규칙**: 한부모 여부는 **한부모 전용 방의 멤버 목록/채팅 안에서만** 표시된다. 일반 방·외부 프로필에는 노출하지 않는다. 가입 시 토글이며 본인이 직접 수정 불가 (정정 필요 시 운영자 문의).

#### 아이 정보 (복수 등록 가능)

| 필드 | 필수 | 설명 |
|------|------|------|
| 태명/별명 | O | 공개용 이름 |
| 생년월 | O | 개월수 자동 계산 |
| 성별 | X | 남/여/비공개 |

> 매칭 시 등록된 모든 아이의 개월수를 고려한다. 둘째/셋째가 있는 가정도 자연스럽게 매칭되도록 "한 명이라도 범위에 들면 매칭" 규칙.

### 2.5 채팅 (Firebase 기반)

| 기능 | 설명 |
|------|------|
| 방 채팅 | 방 참여자 간 그룹 채팅 (Firestore 실시간) |
| 1:1 채팅 | 부모 간 개별 대화 |
| 알림 | 새 메시지 → FCM 푸시 알림 |
| 사진 전송 | 이미지 전송 (Firebase Storage) |

- 채팅 데이터는 Firestore에 저장 (실시간 리스너로 메시지 수신)
- NestJS 서버에서 직접 WebSocket 관리하지 않음
- 서버는 Firestore Admin SDK로 채팅방 생성/삭제/권한 관리만 담당

### 2.6 신뢰 시스템 (MVP 핵심)

| 기능 | 설명 |
|------|------|
| 본인 인증 | KCP 휴대폰 본인확인 (필수, 가입 직후 강제 단계) |
| 모임 후기 + 매너 온도 | 모임 종료 후 **7일 이내** 참여자 간 매너 점수(1~5) + 후기 등록/수정. 7일 이후 점수 고정 |
| 노쇼 추적 | 방장이 모임 후 출석 체크 → 노쇼 횟수가 프로필에 누적. 일정 횟수 이상 시 자동 제재 |
| 위치 노출 단계화 | 방 목록·지도에는 **동 단위**만 노출. 정확한 주소/장소명은 **참여 확정자에게만** 공개 |
| 신고 | 부적절 유저/방/메시지 신고 → 어드민 검토 큐로 이관 |
| 차단 | 특정 유저 차단 → 차단한 유저가 만든 방·메시지 자동 숨김 (양방향) |

#### 매너 온도 산출

- 기본 점수: **36.5°C** (기본값)
- 후기마다 ±0.1~0.5°C 가변. 신고 누적 시 추가 감점
- 프로필에 온도와 함께 "친절했어요 / 약속 잘 지켜요 / 아이와 잘 놀아줬어요" 같은 정성 태그 노출

#### 노쇼 정책

| 조건 | 결과 |
|------|------|
| 모임 시작 24시간 이전 취소 | 패널티 없음 |
| 24시간 이내 취소 | 노쇼 카운트 +0.5 |
| 무단 불참 | 노쇼 카운트 +1 |
| 누적 3회 | 신규 방 참여 7일간 제한 |
| 누적 5회 | 영구 제한 (어드민 검토 후 해제 가능) |

### 2.7 방 참여 조건 — 성별 / 한부모 필터

핵심 차별 기능: **육아 모임 특성상 동질감이 매칭의 핵심**. 두 개의 독립 축으로 제어한다.

| 축 | 옵션 | 설명 |
|----|------|------|
| 참여 성별 | 전체 / 엄마만 / 아빠만 | 방장이 방 생성 시 선택 |
| 한부모 우대 | 무관 / 한부모만 | 한부모 가정 부모만 입장 가능한 전용 모임 |

#### 조합으로 만들어지는 방 카테고리 (6가지)

1. **일반 방** — 전체 + 무관
2. **엄마 전용 방** — 엄마만 + 무관
3. **아빠 전용 방** — 아빠만 + 무관
4. **한부모 통합 방** — 전체 + 한부모만
5. **엄마 한부모 방** — 엄마만 + 한부모만
6. **아빠 한부모 방** — 아빠만 + 한부모만

> **자격 검증 규칙**: 서버에서 방 참여 신청 시 신청자의 `parentGender`, `isSingleParent` 값을 방 조건과 매칭하여 차단. 한부모 전용 방의 멤버 목록/채팅 내부에서만 "한부모" 배지가 노출되고, 그 외 모든 화면에서는 비공개.

### 2.8 번개 모임 (Flash Meeting)

| 항목 | 설명 |
|------|------|
| 정의 | **오늘** 또는 **+24시간 이내** 시작하는 즉시 모임 |
| 일반 방과의 차이 | 30일 이내 제약을 풀고 "오늘"·"지금" 옵션 활성화. 홈 상단에 "⚡ 지금 모이는 방" 섹션 분리 |
| 푸시 우선순위 | 동 단위 매칭 유저에게 즉시 푸시 ("내 동네에서 1시간 후 번개가 열렸어요") |
| 표시 | 일반 방 목록에서 ⚡ 배지로 구분 |
| 노쇼 패널티 | 일반 방과 동일하게 적용 (24시간 이내 취소이므로 카운트 +0.5) |

### 2.9 카카오톡 공유

| 항목 | 설명 |
|------|------|
| SDK | Kakao Share SDK (Custom Template) |
| 공유 카드 | 방 대표 이미지 + 제목 + 날짜/장소 + "참여하기" 버튼 |
| 딥링크 | `kids://room/:id` — 카드 탭 시 앱 설치 여부 확인 후 방 상세로 이동 |
| 사용 위치 | 방 상세 우상단 공유 버튼, 방 생성 완료 후 "친구에게 알리기" 모달 |
| 그로스 의도 | 친한 부모 그룹을 자연 유입시키는 1차 채널. 번개 모임에서 특히 강조 |

### 2.10 발달 가이드 (Growth Guide)

| 항목 | 설명 |
|------|------|
| 정의 | 내 아이 개월수 기준 발달 정보(언어/신체/사회성) + **이번 달 추천 모임/장소** |
| 진입 | 홈 상단 위젯("우리 아이 OO개월차 발달 포인트") + 마이페이지 |
| 데이터 | 0~72개월 월령별 가이드 콘텐츠 (운영자가 등록, CMS에서 관리) |
| 추천 모임 매칭 | 가이드 항목 → 관련 태그 모임 자동 추천 (예: "대근육 발달 시기" → #놀이터, #산책 태그 모임) |
| 차별화 의도 | "모임 안 잡아도 앱을 여는 이유" — 콘텐츠 hook |

### 2.11 팔로우 / 단골 부모

| 항목 | 설명 |
|------|------|
| 정의 | 마음 맞는 부모를 팔로우 → 그 부모가 새 방을 만들면 푸시 알림 |
| 진입 | 유저 프로필 상단 "팔로우" 버튼, 모임 종료 후 매너 평가 화면 하단 "단골로 등록" 토글 |
| 비공개 | 팔로워/팔로잉 목록은 **본인만** 조회 가능 (스토킹 방지) |
| 1:1 채팅 대체 | MVP에서 1:1 채팅 없이도 관계 유지 가능 (다음 모임에서 다시 만남) |

### 2.12 준비물 체크리스트

| 항목 | 설명 |
|------|------|
| 정의 | 방 생성 시 추천 준비물 토글 (기저귀, 물티슈, 간식, 돗자리, 여벌 옷 등) |
| 표시 | 방 상세 하단 "준비물" 카드. 참여 확정 시 푸시로 1회 안내 |
| 운영자 관리 | 기본 항목은 운영자가 CMS에서 관리. 커스텀 입력도 가능 |

### 2.13 알림

| 알림 유형 | 설명 |
|------------|------|
| 참여 신청 알림 | 내 방에 누가 신청했을 때 (방장) |
| 수락/거절 알림 | 신청 결과 (신청자) |
| 모임 리마인더 | 모임 1시간 전, 하루 전 |
| 채팅 알림 | 새 메시지 |
| 새 방 알림 | 내 조건에 맞는 새 방이 올라왔을 때 |
| 번개 모임 알림 | 동 단위 매칭 유저에게 즉시 푸시 |
| 후기 요청 알림 | 모임 종료 직후 + 종료 6일 후 (마감 1일 전) 리마인드 |
| 팔로우 새 방 알림 | 팔로우한 부모가 새 방을 만들었을 때 |
| 발달 가이드 알림 | 우리 아이가 새로운 개월수에 진입하면 월 1회 |
| 노쇼 경고 알림 | 노쇼 카운트 누적 시 경고 |

## 3. 화면 구성 (Screen Flow)

### 3.1 메인 탭 구조

```
[홈] [검색] [방 만들기(+)] [채팅] [마이페이지]
```

### 3.2 주요 화면 목록

| 화면 | 설명 |
|------|------|
| 스플래시 | 앱 로딩 |
| 온보딩 | 서비스 소개 (3페이지) |
| 로그인 | 소셜 로그인 (카카오, Apple, Google) + 이메일 로그인 |
| 회원가입 | 프로필 + 아이 정보 + 핸드폰 본인 인증 |
| 홈 | 내 지역 기반 추천 방 목록 + 지도 뷰 전환 |
| 지도 | 네이버 지도 (클러스터/핀 뷰) |
| 방 상세 | 방 정보 + 참여자 목록 + 참여 신청 |
| 방 생성 | 새 방 만들기 폼 |
| 검색 | 필터 기반 방 검색 |
| 참여 관리 | 방장용 - 신청자 수락/거절 |
| 채팅 목록 | 참여중인 방 채팅 + 1:1 채팅 |
| 채팅방 | 메시지 주고받기 |
| 마이페이지 | 프로필 / 내 모임 / 후기 / 설정 |
| 프로필 수정 | 내 정보 + 아이 정보 수정 |
| 유저 프로필 | 다른 유저 정보 조회 |
| 알림 | 알림 목록 |

### 3.3 유저 플로우

```
[첫 방문]
앱 설치 → 온보딩 → 소셜 로그인 → 프로필 설정 → 아이 정보 입력 → 홈

[방 참여]
홈/검색 → 방 발견 → 방 상세 확인 → 참여 신청 → (자유입장: 즉시 참여 / 승인: 대기) → 채팅방 입장

[방 생성]
+ 버튼 → 방 생성 폼 → 등록 → 신청자 대기 → 수락/거절 → 모임 진행

[모임 완료]
모임 시간 경과 → 방 상태 "완료" → 후기 작성 요청 알림 → 후기 작성
```

## 5. MVP 범위 (1단계)

> 핵심만 빠르게 만들어서 반응 확인

### MVP 포함

#### 인증/프로필
- [x] 소셜 로그인 (카카오, Apple, Google) + 이메일 로그인
- [x] KCP 핸드폰 본인 인증
- [x] 프로필 + 아이 정보 등록 (복수 아이 매칭 포함)
- [x] **부모 성별 + 한부모 가정 토글** (가입 시 1회, 수정 불가)

#### 방
- [x] 방 생성 (지역, 날짜, 시간, 개월수, 장소, 인원, 입장방식, **성별 필터, 한부모 우대, 번개 모임, 준비물 체크리스트**)
- [x] 방 목록 조회 (지역/개월수 필터, 약속일 지난 방 자동 제외)
- [x] 네이버 지도 뷰 (클러스터 + 핀)
- [x] **위치 노출 단계화** (목록=동, 정확한 주소=참여 확정 후)
- [x] 방 상세 보기
- [x] 참여 신청 (자유입장 + 승인)
- [x] **번개 모임** (24시간 이내 즉시 모임)
- [x] 방 채팅 (NestJS WebSocket Gateway 기반)

#### 신뢰/안전 (육아 모임 필수)
- [x] **신고/차단** (유저/방/메시지)
- [x] **후기 + 매너 온도** (모임 종료 후 7일 이내 등록·수정)
- [x] **노쇼 추적** (방장 출석 체크 + 자동 제재)

#### 그로스/리텐션
- [x] **카카오톡 공유** (방 미리보기 카드 + 딥링크)
- [x] **팔로우 / 단골 부모** (새 방 알림)
- [x] **발달 가이드** (개월수별 콘텐츠 + 추천 모임 매칭)

#### 기본
- [x] 푸시 알림 (참여 신청, 수락/거절, 번개, 후기 요청, 팔로우 새 방)
- [x] 마이페이지 (내 모임 목록, 매너 온도, 노쇼 카운트)

### MVP 제외 (2단계 이후)

- [ ] 1:1 채팅
- [ ] 사진 전송 (S3 업로드)
- [ ] 모임 인증샷 / 후기 사진
- [ ] 육아 크루 (고정 정기 그룹)
- [ ] 장소 추천/예약 연동
- [ ] 커뮤니티 게시판
- [ ] 장소 리뷰

|------|------|------|
