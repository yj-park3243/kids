## 4.1 서버 Cron 작업

> NestJS `@nestjs/schedule` 기반. 신규 기능 도입으로 다음 작업이 PM2 단일 인스턴스에서 동작한다.

| Cron | 주기 | 동작 | 모듈 |
|------|------|------|------|
| `room-status-transition` | 매 5분 | `RECRUITING`→`IN_PROGRESS`→`COMPLETED` 전환 (기존) | room |
| `review-request-dispatch` | 방 종료 직후 + 매 1시간 (D+6 검사) | 종료된 방 멤버에게 `REVIEW_REQUEST` 푸시. 마감 1일 전 리마인드 | room-scheduler |
| `no-show-release` | 매일 새벽 4시 | `can_join_at < now()` 유저의 `can_join_at` NULL 처리, 카운트는 유지 | user/no-show |
| `growth-month-update` | 매월 1일 02:00 | 자녀 개월수 변동 유저에게 `GROWTH_UPDATE` 푸시 | growth-guide |
| `flash-meeting-broadcast` | 방 생성 이벤트 즉시 | `isFlashMeeting=true` 방 생성 시 동 + 개월수 매칭 유저 조회 → `NEW_FLASH` FCM 멀티캐스트 | room |
| `room-reminder` | 매 10분 | 모임 1시간 전 / 24시간 전 `ROOM_REMINDER` (기존) | room |

## 7. 배포 방식

| 항목 | 방식 |
|------|------|
| 서버 배포 | 로컬 `server/deploy.sh` → rsync로 EC2 전송 → 원격에서 `npm install` + `npm run build` → `pm2 restart` (`ecosystem.config.cjs`) |
| 앱 배포 | `app/deploy.sh` (fastlane 래퍼) — `fastlane deploy` (App Store/Play 정식 트랙), `fastlane internal` (TestFlight / Play 내부 트랙) |
| 어드민 배포 | `admin/deploy.sh` — Vite `npm run build` → `admin.growtogether.kr` Nginx 정적 루트로 업로드 |
| DB 마이그레이션 | TypeORM migration:run |
