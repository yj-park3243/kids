# 같이크자 앱 테스트 시나리오

> 기획서(`01_기획서.md`)·기능정의서(`02_기능정의서.md`)를 기준으로, 앱 전 기능에 대한 QA 테스트 시나리오를 정리한다.
> 자동화 e2e(`app/integration_test/`)가 이미 커버하는 핵심 루프는 [부록 A](#부록-a-자동화-e2e-매핑)에서 매핑한다.

---

## 0. 개요

### 0.1 표기법

| 표기 | 의미 |
|------|------|
| **케이스 ID** | `{영역}-{유형}-{번호}` (예: `AUTH-REG-01`) |
| **우선순위** | `P0` 핵심 루프·안전(필수) / `P1` 중요 / `P2` 부가·엣지 |
| **자동화** | 🤖 자동(integration_test) / 🖐 수동 / ⚙ 혼합(API 자동+UI 수동) |

### 0.2 테스트 계정 / 환경

| 역할 | parentGender | isSingleParent | 자녀 | 용도 |
|------|--------------|----------------|------|------|
| **A** | MOM | true | 2명 | 방장 — 생성/관리/출석/후기 |
| **B** | MOM | true | 2명 | 참여자(한부모) — 전 라운드 입장 가능 |
| **C** | DAD | false | 4명 | 거부 케이스 — MOM_ONLY/한부모 방 거부, 차단/신고 |
| **D** | DAD | false | 0명 | 자녀 없음·노쇼·정지 등 음성 케이스용(신규 제안) |

- **서버**: `https://api.growtogether.kr/v1` (운영). 더미 누적 방지를 위해 **stg/e2e 별도 환경 권장**.
- **디바이스**: iOS 시뮬레이터 ≥ 2대(실시간 채팅·승인 플로우 검증). 푸시·딥링크·KCP 웹뷰는 **실기기** 필요.
- **본인인증 우회**: e2e는 `UPDATE "user" SET is_phone_verified=true` (orchestrator). 수동 QA는 KCP 실제 인증 1회 수행.

## 부록 A. 자동화 e2e 매핑

현재 `app/integration_test/main_scenario_test.dart`는 **시뮬레이터 3대(A/B/C) × 3 라운드**로 핵심 루프를 자동 검증한다. 본 문서의 🤖 케이스 ↔ e2e 라운드 매핑:

| e2e 위치 | 커버 케이스 |
|----------|-------------|
| Round 1 (FREE/ALL) | ROOM-C-01, ROOM-J-01, CHAT-03/04, REV-01, ROOM-ST-05 |
| Round 2 (APPROVAL/MOM_ONLY) | ROOM-J-05, **ROOM-J-06**, ROOM-J-08, ROOM-REJ-01 |
| Round 3 (FREE/한부모) | ROOM-CAT-04, ROOM-REJ-02 |
| **Round 4 — B** (팔로우/위치) | FOL-01, FOL-03, ROOM-LOC-03 |
| **Round 4 — C** (신고/위치/권한) | REP-01, ROOM-LOC-02, NS-03 |
| **Round 5 — A** (노쇼/후기) | NS-01, REV-04 |
| **Round 6 — A** (프로필/조회) | AUTH-TOK-04, MY-05, MY-10 |

> Round 4~6은 본 작업으로 `main_scenario_test.dart`에 추가했다(부작용 없는 자동화 가능분).
> **Round 2 승인 처리(ROOM-J-06)는 기존 헬퍼가 잘못된 페이로드 `{status:'APPROVED'}`를 보내 `forbidNonWhitelisted`로 400 거부되던 버그를 정식 `{action:'ACCEPT'}`로 수정**하면서 비로소 실제로 통과하게 됐다.
> KCP 인증·지도·카카오 공유·푸시·실시간 소켓 수신·UI 위젯 탭은 외부 SDK/디바이스 의존으로 **수동(🖐) 유지**.

### 자동화 우선 확장 후보(미구현)

1. **BLK-* 차단 양방향** — 서버가 차단 시 공유 방 멤버십·상호 팔로우를 삭제(`removeSharedRoomMembership`)하므로 현 3-user 공유방 셋업에서는 다른 역할의 후기 작성과 충돌한다. **차단 전용 격리 계정/방**을 orchestrator에 추가하면 100% 자동화 가능
2. **NFR-03 정원 동시성** — 동일 방 마지막 자리에 2 토큰 동시 join → 1 성공/1 거부 단언
3. **REV-06 7일 경과 고정** — orchestrator가 `completed_at`을 8일 전으로 SQL 주입 후 후기 거부 단언
4. **ROOM-REJ-04/05 노쇼 제재** — `no_show_count` SQL 주입 후 join 거부 단언
5. **ROOM-L-02 지난 방 제외 / ROOM-L-08 차단 방 숨김** — `date` 과거 방·차단 관계 SQL 주입 후 목록 미노출 단언
6. **ROOM-M-03 강퇴** — `kickMember` 헬퍼는 추가됨. 격리 멤버 대상이면 자동화 가능
