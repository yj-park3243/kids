# 데이터베이스 설계서

## DBMS: PostgreSQL
## ORM: TypeORM

---

## ERD 관계 요약

```
[PostgreSQL]
User ──1:N── Child
User ──1:N── Room (host)
User ──1:N── RoomMember
User ──1:N── JoinRequest
User ──1:N── Notification
User ──1:N── DeviceToken
User ──1:N── ChatMessage (sender)

Room ──1:N── RoomMember
Room ──1:N── JoinRequest
Room ──1:N── ChatMessage      # Room.id 가 곧 채팅방 ID
```

> 채팅은 별도 채팅방 엔티티 없이 `Room` 자체를 채팅방으로 사용한다.
> 실시간 송수신은 NestJS `ChatGateway` (Socket.IO, namespace `/chat`).

## 1. User (유저)

```sql
CREATE TABLE "user" (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  auth_provider       VARCHAR(20) NOT NULL,          -- KAKAO, APPLE, GOOGLE, EMAIL
  social_id           VARCHAR(100) UNIQUE,            -- 소셜 로그인 고유 ID (EMAIL이면 null)
  email               VARCHAR(255) UNIQUE,            -- 이메일 (소셜이면 null 가능)
  password_hash       VARCHAR(255),                   -- 이메일 로그인 시 bcrypt 해시 (소셜이면 null)
  nickname            VARCHAR(20) UNIQUE,             -- 프로필 설정 전에는 null
  profile_image_url   TEXT,
  introduction        VARCHAR(200),
  region_sido         VARCHAR(20),
  region_sigungu      VARCHAR(20),
  region_dong         VARCHAR(20),
  is_profile_complete BOOLEAN DEFAULT false,
  is_phone_verified   BOOLEAN DEFAULT false,          -- 핸드폰 본인 인증 완료 여부
  phone_number        VARCHAR(20),                    -- 인증된 전화번호 (마스킹 저장)
  is_admin            BOOLEAN DEFAULT false,          -- 관리자 여부 (Admin API 접근)
  status              VARCHAR(20) DEFAULT 'ACTIVE',   -- ACTIVE, WITHDRAWN, BANNED
  withdrawn_at        TIMESTAMP,
  created_at          TIMESTAMP DEFAULT now(),
  updated_at          TIMESTAMP DEFAULT now()
);
```

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| auth_provider | VARCHAR(20) | 인증 방식 (KAKAO, APPLE, GOOGLE, EMAIL) |
| social_id | VARCHAR(100) | 소셜 고유 ID (이메일 가입 시 null) |
| email | VARCHAR(255) | 이메일 (소셜 가입 시 null 가능) |
| password_hash | VARCHAR(255) | 비밀번호 해시 (소셜 가입 시 null) |
| nickname | VARCHAR(20) | 닉네임 (유니크, 프로필 설정 전에는 null) |
| profile_image_url | TEXT | 프로필 이미지 URL |
| introduction | VARCHAR(200) | 자기소개 |
| region_sido | VARCHAR(20) | 시/도 |
| region_sigungu | VARCHAR(20) | 시/군/구 |
| region_dong | VARCHAR(20) | 읍/면/동 |
| is_profile_complete | BOOLEAN | 프로필 설정 완료 여부 |
| is_phone_verified | BOOLEAN | 핸드폰 본인 인증 여부 |
| phone_number | VARCHAR(20) | 인증된 전화번호 (마스킹) |
| is_admin | BOOLEAN | 관리자 여부 (Admin API 접근 권한) |
| status | VARCHAR(20) | 유저 상태 |
| withdrawn_at | TIMESTAMP | 탈퇴 일시 |
| created_at | TIMESTAMP | 가입 일시 |
| updated_at | TIMESTAMP | 수정 일시 |

**인덱스**
- `UNIQUE (social_id)` — WHERE social_id IS NOT NULL
- `UNIQUE (email)` — WHERE email IS NOT NULL
- `UNIQUE (nickname)`
- `INDEX (region_sido, region_sigungu, region_dong)`

## 2. Child (아이)

```sql
CREATE TABLE child (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES "user"(id),
  nickname    VARCHAR(20) NOT NULL,
  birth_year  INT NOT NULL,
  birth_month INT NOT NULL,
  gender      VARCHAR(10),  -- MALE, FEMALE, null
  created_at  TIMESTAMP DEFAULT now(),
  updated_at  TIMESTAMP DEFAULT now()
);
```

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| user_id | UUID | FK → User |
| nickname | VARCHAR(20) | 태명/별명 |
| birth_year | INT | 출생 연도 |
| birth_month | INT | 출생 월 (1~12) |
| gender | VARCHAR(10) | 성별 (nullable) |

**인덱스**
- `INDEX (user_id)`

**참고**: 개월수는 애플리케이션 레벨에서 계산 → `(현재연도 - birth_year) * 12 + (현재월 - birth_month)`

## 3. Room (방/모임)

```sql
CREATE TABLE room (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  host_id         UUID NOT NULL REFERENCES "user"(id),
  title           VARCHAR(60) NOT NULL,
  description     VARCHAR(1000) NOT NULL,
  region_sido     VARCHAR(20) NOT NULL,
  region_sigungu  VARCHAR(20) NOT NULL,
  region_dong     VARCHAR(20) NOT NULL,
  date            DATE NOT NULL,
  start_time      TIME NOT NULL,
  end_time        TIME,
  age_month_min   INT NOT NULL,
  age_month_max   INT NOT NULL,
  place_type      VARCHAR(20) NOT NULL,  -- PLAYGROUND, KIDS_CAFE, PARTY_ROOM, PARK, OTHER
  place_name      VARCHAR(100),
  place_address   VARCHAR(200),
  latitude        DOUBLE PRECISION,       -- 위도 (Geocoding 결과)
  longitude       DOUBLE PRECISION,       -- 경도 (Geocoding 결과)
  max_members     INT NOT NULL,
  current_members INT DEFAULT 1,
  join_type       VARCHAR(20) NOT NULL,  -- FREE, APPROVAL
  cost            INT DEFAULT 0,
  cost_description VARCHAR(200),
  tags            TEXT[],                 -- PostgreSQL 배열
  status          VARCHAR(20) DEFAULT 'RECRUITING',  -- RECRUITING, CLOSED, IN_PROGRESS, COMPLETED, CANCELLED
  chat_room_id    UUID,
  created_at      TIMESTAMP DEFAULT now(),
  updated_at      TIMESTAMP DEFAULT now()
);
```

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| host_id | UUID | FK → User (방장) |
| title | VARCHAR(60) | 제목 |
| description | VARCHAR(1000) | 설명 |
| region_* | VARCHAR | 지역 (시/구/동) |
| date | DATE | 모임 날짜 |
| start_time | TIME | 시작 시간 |
| end_time | TIME | 종료 시간 (nullable) |
| age_month_min | INT | 최소 개월수 |
| age_month_max | INT | 최대 개월수 |
| place_type | VARCHAR(20) | 장소 유형 enum |
| place_name | VARCHAR(100) | 장소명 |
| place_address | VARCHAR(200) | 상세 주소 |
| latitude | DOUBLE PRECISION | 위도 (네이버 Geocoding) |
| longitude | DOUBLE PRECISION | 경도 (네이버 Geocoding) |
| max_members | INT | 최대 인원 |
| current_members | INT | 현재 인원 |
| join_type | VARCHAR(20) | 입장 방식 |
| cost | INT | 비용 (0 = 무료) |
| cost_description | VARCHAR(200) | 비용 설명 |
| tags | TEXT[] | 태그 배열 |
| status | VARCHAR(20) | 방 상태 |
| chat_room_id | UUID | FK → ChatRoom |

**인덱스**
- `INDEX (host_id)`
- `INDEX (region_sido, region_sigungu, region_dong)`
- `INDEX (date)`
- `INDEX (status)`
- `INDEX (age_month_min, age_month_max)`
- `INDEX (latitude, longitude)` ← 지도 영역 조회용
- `COMPOSITE INDEX (region_dong, date, status)` ← 주요 검색용

## 4. RoomMember (방 참여자)

```sql
CREATE TABLE room_member (
  id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  room_id   UUID NOT NULL REFERENCES room(id),
  user_id   UUID NOT NULL REFERENCES "user"(id),
  is_host   BOOLEAN DEFAULT false,
  joined_at TIMESTAMP DEFAULT now(),

  UNIQUE (room_id, user_id)
);
```

**인덱스**
- `UNIQUE (room_id, user_id)`
- `INDEX (user_id)`

## 5. JoinRequest (참여 신청)

```sql
CREATE TABLE join_request (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  room_id     UUID NOT NULL REFERENCES room(id),
  user_id     UUID NOT NULL REFERENCES "user"(id),
  status      VARCHAR(20) DEFAULT 'PENDING',  -- PENDING, ACCEPTED, REJECTED, CANCELLED
  created_at  TIMESTAMP DEFAULT now(),
  updated_at  TIMESTAMP DEFAULT now(),

  UNIQUE (room_id, user_id)
);
```

**인덱스**
- `UNIQUE (room_id, user_id)`
- `INDEX (room_id, status)`

## 6~8. 채팅 (Firebase Firestore)

> 채팅 관련 데이터(ChatRoom, ChatRoomUser, ChatMessage)는 **Firebase Firestore**에 저장.
> PostgreSQL에 별도 테이블 없음.

### Firestore 구조

```
chatRooms/{chatRoomId}
├── roomId: string          (PostgreSQL Room.id 참조)
├── memberIds: string[]     (참여자 User.id 배열)
├── lastMessage: string
├── lastMessageAt: timestamp
├── createdAt: timestamp
│
└── messages/{messageId}    (subcollection)
    ├── senderId: string
    ├── senderNickname: string
    ├── content: string
    ├── type: "TEXT" | "SYSTEM" | "IMAGE"
    └── createdAt: timestamp
```

### Room 테이블과의 연결

- Room 테이블의 `chat_room_id` 컬럼에 Firestore chatRoom 문서 ID 저장
- NestJS에서 방 생성 시 Firestore Admin SDK로 chatRoom 문서 생성 → 문서 ID를 Room.chat_room_id에 저장

## 9. Notification (알림)

```sql
CREATE TABLE notification (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES "user"(id),
  type        VARCHAR(30) NOT NULL,  -- JOIN_REQUEST, JOIN_ACCEPTED, JOIN_REJECTED, ROOM_CANCELLED, ROOM_REMINDER, NEW_CHAT, NEW_ROOM
  title       VARCHAR(100) NOT NULL,
  body        VARCHAR(300) NOT NULL,
  data        JSONB,                 -- { roomId, chatRoomId 등 네비게이션 데이터 }
  is_read     BOOLEAN DEFAULT false,
  created_at  TIMESTAMP DEFAULT now()
);
```

**인덱스**
- `INDEX (user_id, created_at DESC)`
- `INDEX (user_id, is_read)`

## 12. ChatMessage (채팅 메시지)

```sql
CREATE TABLE chat_message (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  room_id         UUID NOT NULL REFERENCES room(id) ON DELETE CASCADE,  -- Room.id == 채팅방 ID
  sender_id       UUID REFERENCES "user"(id) ON DELETE SET NULL,        -- 시스템 메시지는 NULL
  sender_nickname VARCHAR(40) NOT NULL,                                 -- 전송 시점 닉네임 스냅샷
  content         TEXT NOT NULL,
  type            VARCHAR(10) NOT NULL DEFAULT 'TEXT',                  -- TEXT, SYSTEM, IMAGE
  created_at      TIMESTAMP DEFAULT now()
);
```

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| room_id | UUID | 방 ID (Room.id == 채팅방) |
| sender_id | UUID | 발신자 (SYSTEM 이면 NULL) |
| sender_nickname | VARCHAR(40) | 전송 시점 닉네임 스냅샷 |
| content | TEXT | 메시지 본문 (최대 1000자 - DTO 제약) |
| type | VARCHAR(10) | TEXT / SYSTEM / IMAGE |
| created_at | TIMESTAMP | 전송 일시 |

**인덱스**
- `INDEX (room_id, created_at)` — 방별 시간 역순 조회 최적화

**전달 경로**
1. 앱 → `POST /v1/chat/rooms/:roomId/messages` (REST) 로 저장
2. 서버가 저장 후 `ChatGateway` 로 `room:{roomId}` 네임스페이스에 Socket.IO `message` 이벤트 브로드캐스트
3. 앱은 `/chat` namespace WebSocket 으로 구독

## TypeORM Entity 네이밍 규칙

| 항목 | 규칙 | 예시 |
|------|------|------|
| Entity 클래스 | PascalCase | `RoomMember` |
| 테이블명 | snake_case | `room_member` |
| 컬럼명 | snake_case (DB) / camelCase (Entity) | DB: `host_id` / Entity: `hostId` |
| 관계 | camelCase | `room.members`, `user.children` |
