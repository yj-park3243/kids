# 기능 정의서

## 1. 인증 (Auth)

### 1.1 소셜 로그인

| 항목 | 내용 |
|------|------|
| 기능 ID | AUTH-001 |
| 설명 | 소셜 OAuth2.0을 통한 로그인/회원가입 |
| 지원 | 카카오, Apple, Google |
| 트리거 | 로그인 화면에서 소셜 로그인 버튼 탭 |
| 선행 조건 | 없음 |
| 정상 플로우 | 1. 소셜 SDK 인증 → 2. 서버에 소셜 토큰 + provider 전송 → 3. 서버에서 소셜 사용자 정보 조회 → 4. 신규 유저면 회원가입 플로우, 기존이면 JWT 발급 → 5. 홈 이동 |
| 예외 | 인증 취소 → 로그인 화면 유지 |
| 결과 | Access Token + Refresh Token 발급, 로컬 저장 |

### 1.2 이메일 로그인

| 항목 | 내용 |
|------|------|
| 기능 ID | AUTH-001-2 |
| 설명 | 이메일 + 비밀번호로 로그인/회원가입 |
| 회원가입 플로우 | 1. 이메일 입력 → 2. 이메일 중복 체크 → 3. 비밀번호 설정 (8자 이상, 영문+숫자+특수문자) → 4. 서버에 등록 → 5. JWT 발급 → 6. 프로필 설정 이동 |
| 로그인 플로우 | 1. 이메일 + 비밀번호 입력 → 2. 서버 인증 → 3. JWT 발급 → 4. 홈 이동 |
| 비밀번호 찾기 | 이메일로 비밀번호 재설정 링크 발송 |

## 2. 방 (Room)

### 2.1 방 생성

| 항목 | 내용 |
|------|------|
| 기능 ID | ROOM-001 |
| 설명 | 새 육아 모임 방 생성 |

#### 입력 필드 상세

| 필드 | 타입 | 필수 | 유효성 |
|------|------|------|--------|
| title | string | O | 2~30자 |
| description | string | O | 10~500자 |
| regionSido | string | O | 시/도 |
| regionSigungu | string | O | 시/군/구 |
| regionDong | string | O | 읍/면/동 |
| date | date | O | 오늘 이후, 30일 이내 |
| startTime | time | O | - |
| endTime | time | X | startTime 이후 |
| ageMonthMin | int | O | 0~84 |
| ageMonthMax | int | O | ageMonthMin 이상 |
| placeType | enum | O | PLAYGROUND, KIDS_CAFE, PARTY_ROOM, PARK, OTHER |
| placeName | string | X | 장소명 |
| placeAddress | string | X | 상세 주소 |
| maxMembers | int | O | 2~10 |
| joinType | enum | O | FREE(자유입장), APPROVAL(승인필요) |
| cost | int | X | 0 이상 (원 단위) |
| costDescription | string | X | 비용 설명 |
| tags | string[] | X | 최대 5개, 각 10자 이내 |

#### 결과

- 방 생성, 상태 = RECRUITING
- 방장 자동 참여 (참여 인원 1명)
- 방 채팅방 자동 생성

### 2.2 방 목록 조회

| 항목 | 내용 |
|------|------|
| 기능 ID | ROOM-002 |
| 설명 | 조건에 맞는 방 목록 조회 |
| 기본 정렬 | 날짜 가까운 순 |
| 페이징 | 커서 기반, 20개씩 |
| **핵심 규칙** | **약속 날짜(date)가 오늘 이전인 방은 목록에서 자동 제외** |

> 서버 쿼리에서 `WHERE date >= CURRENT_DATE` 조건 필수 적용. 지난 모임은 `GET /rooms/my?status=PAST`로만 조회 가능.

#### 필터 조건

| 필터 | 타입 | 기본값 |
|------|------|--------|
| region | string | 내 지역 (동 단위) |
| date | date/range | 오늘 이후 |
| ageMonth | int | 내 첫째 아이 개월수 ±3개월 |
| placeType | enum | 전체 |
| joinType | enum | 전체 |
| costFree | boolean | false |

#### 목록 아이템 표시 정보

```

날짜 시간 | 지역(동)
대상: N~M개월 | 참여 X/Y명
비용: 무료 or N원
태그들
```

### 2.3 네이버 지도 뷰

| 항목 | 내용 |
|------|------|
| 기능 ID | ROOM-007 |
| 설명 | 방 목록을 네이버 지도 위에서 탐색 |
| SDK | Naver Map SDK (flutter_naver_map) |

#### 지도 줌 레벨별 표시

| 줌 레벨 | 표시 방식 | 동작 |
|---------|-----------|------|
| ~13 (축소/광역) | **클러스터** | 동 단위로 묶어서 원형 뱃지에 방 개수 표시. 원 크기는 개수에 비례 |
| 14~ (확대/상세) | **핀 마커** | 개별 방의 약속 장소에 핀 마커 표시 |

#### 핀 마커 탭 동작

1. 핀 탭 → 하단에 방 요약 카드 팝업 (제목, 날짜, 개월수, 인원)
2. 카드 탭 → 방 상세 화면 이동

#### 좌표 관리

- 방 생성 시 장소 주소 → 네이버 Geocoding API로 위도/경도 변환
- Room 테이블에 `latitude`, `longitude` 저장
- 지도 뷰 API: 화면 영역(SW/NE 좌표) 기반으로 해당 영역 내 방 조회

#### 클러스터 집계 API

- 서버에서 동(dong) 단위로 GROUP BY → 동별 방 개수 + 대표 좌표 반환
- 약속 날짜 지난 방은 집계에서 제외


## 3. 채팅 (Chat) — Firebase Firestore 기반

### 3.1 아키텍처

```
Flutter App ←→ Firestore (실시간 리스너)
NestJS Server → Firestore Admin SDK (채팅방 생성/삭제/권한)
```

- 채팅 메시지 읽기/쓰기: **Flutter → Firestore 직접 연결** (실시간)
- 채팅방 생성/삭제/멤버 관리: **NestJS → Firestore Admin SDK**
- Firestore Security Rules로 권한 제어 (참여자만 읽기/쓰기)

### 3.2 Firestore 구조

```
chatRooms/{chatRoomId}
├── roomId: string (모임 방 ID)
├── memberIds: string[] (참여자 ID 배열)
├── lastMessage: string
├── lastMessageAt: timestamp
├── createdAt: timestamp
│
└── messages/{messageId}
    ├── senderId: string
    ├── senderNickname: string
    ├── content: string
    ├── type: "TEXT" | "SYSTEM" | "IMAGE"
    ├── createdAt: timestamp
```
